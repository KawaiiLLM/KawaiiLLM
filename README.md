### 1. 项目简介
KawaiiLLM 是一个构建 ACGN（动画、漫画、游戏、小说）领域大语言模型（LLM）的项目。
核心目标有两个：
1. 实现ACGN风格的角色扮演。
2. 实现记忆压缩。
主要流程：
- 继续预训练：收集ACGN领域语料（主要是中文、日语、英语），训练Encoder编码高维语义信息为抽象记忆的能力，训练LLM读取抽象记忆并学习ACGN领域知识与语言风格。
- 后训练（未确定）：规范LLM角色扮演的输出格式（如思维链内为心理活动）、理解多组记忆输入。
- 强化学习（未确定）：训练记忆的动态读取、角色扮演的优化
### 2. 模型架构
### 2.1 Encoder
编码器（通常是较小的Embedder）负责将长文本（500-4k tokens）压缩为高维隐向量。
维特根斯坦认为，很多内在体验（如微妙的情绪、复杂的直觉）是语言无法表达的。在大模型中，这些“不可言说”的东西往往对应着潜在空间中某些难以被Token化的特征方向。向量往往包含了人类语言无法描述的模式。所以这个过程相比“文本压缩”更像是“感觉还原”，即把文本还原为LLM的“体验”，就像多模态LLM把图片还原为视觉体验。
#### 2.2 Decoder
解码器（通常是较大的LLM）可以接收并读取编码器输出的高维隐向量，从中获得读到原始文章后的“体验”。
#### 2.3 Projector
- 将编码器的隐藏状态映射到解码器的维度
- 作为编码器-解码器之间的桥梁，允许两者拥有不同的隐藏层大小

#### 3.1.2 数据格式
合并准备好的数据，最终数据格式：
```json
[
  {"source": "bilibili", "id": "84", split: "0", "text": "document"},
  {"source": "bilibili", "id": "84", split: "1", "text": "document"},
  {"source": "moegirl", "id": "0", split: "0", "text": "document"},
  {"source": "moegirl", "id": "0", split: "1", "text": "document"},
  {"source": "novels", "id": "11860530", split: "0", "text": "document"},
  {"source": "novels", "id": "11860530", split: "1", "text": "document"},
  {"source": "novels", "id": "11860530", split: "2", "text": "document"},
  ...
]
```
其中:
- `source`：表示文本来自哪种类型数据
- `id`：文本在该类型数据中的唯一标识，如果有原始 id 则直接复用，否则自建 id。
- `split`：文本切分为多个chunk后，每个chunck 的索引 id，用于索引连贯文本。
- `text`：增量预训练的目标，每个文本块的长度不超过 **4096** tokens。
#### 3.1.3 模型准备
* **Encoder**：
	* [Qwen3-Embedding-8B](https://huggingface.co/Qwen/Qwen3-Embedding-8B) 作为初始模型，拥有较好的语义理解与压缩能力。
* **Projector**：
	* 参考Qwen2.5-VL的两层MLP，MLP Layer 1 → GELU → MLP Layer 2
* **Decoder**：
	* Qwen3-8B-Base 作为初始模型。
	* 备选Qwen3-VL-8B-Instruct，因为训练过理解图片向量输入，可能迁移性更好。
#### 3.1.4 模型训练
##### 数据流
* **Encoder**：输入长文本（4k discrete tokens），输出一组向量（128 latent tokens）。
* **Projector**: 输入 Encoder 输出的向量，输出转换后的向量（与 Decoder 向量维度一致）。
* **Decoder**：输入 Projector 输出的向量，输出自然语言文本。
##### 训练目标
* 通过 NTP 学习领域知识，并学会压缩、读取向量记忆。分为两个任务（参考PCC）：
	* 文本重建：Decoder 还原原始 Encoder 输入长文本中的每个 token。
	* 文本续写：Decoder 续写原始 Encoder 输入长文本的下一连续文本块的每个 token。
* 为实现不同程度的记忆清晰度（压缩率），输入 Decoder 的 Latent token 数量不固定：
	* One Token：仅使用 **Projector** 输出的第一个 latent token 进行训练，强调主题与模糊记忆。
	* Multiple Tokens：使用前几个 latent token 进行训练，逐步加入细节的还原。
##### 训练流程
以下流程参考Qwen2.5-VL。Qwen3-Embedding 已经经过对比学习预训练，无需单独训练。
* ***阶段1：记忆-语言联合训练**
	解冻所有参数（Encoder、Projector、Decoder）进行全面训练。
	* 前期：使用文本重建 + One Token 策略
	* 中后期：慢慢加入文本续写和更多的 Latent Tokens

```python
# 训练过程（简化版）

Step 1: Forward Pass
context → Encoder → [v1, v2, ..., vn] → Projector → [h1, h2, ..., hn] → LLM → output
                ↑ 记忆表示             ↑ 当前映射           ↑ 生成结果

Step 2: Compute Loss
loss = CrossEntropy(output, target)  # 如"傲娇（tsundere，ツンデレ）是一种人物性格..."

Step 3: Backward Pass (关键!)
∂loss/∂LLM → ∂loss/∂Projector → ∂loss/∂Encoder
     ↓              ↓                  ↓
  "需要更好    "需要这样映射"    "需要这种特征!"
   的记忆表示"

Step 4: Update All (同时更新!)
Encoder.weights      -= lr * ∂loss/∂Encoder
Projector.weights -= lr * ∂loss/∂Projector  
LLM.weights      -= lr * ∂loss/∂LLM
```
