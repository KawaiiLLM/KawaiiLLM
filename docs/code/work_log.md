# ACGN 代码数据收集工作日志

**日期**: 2026-02-15
**任务目标**: 从 ModelScope 的 `starcoderdata` 数据集中收集约 1GB 的多语言代码数据，用于 ACGN 领域模型的继续预训练。

## 1. 需求分析与配置
- **总数据量**: ~1GB
- **数据源**: `AI-ModelScope/starcoderdata`
- **语言分布**:
  - Python: 25% (250MB)
  - C++: 20% (200MB)
  - C#: 20% (200MB)
  - Java: 10% (100MB)
  - JavaScript: 10% (100MB)
  - TypeScript: 10% (100MB)
  - Go: 5% (50MB)
  - Rust: 5% (50MB)
  - Lua: 3% (30MB)
  - SQL: 2% (20MB)

## 2. 遇到的挑战与解决方案

### 2.1 ModelScope SDK 子集加载问题
- **问题**: 尝试使用 `subset_name='python'` 加载特定语言子集时失败，API 返回不支持该子集。
- **尝试**: 尝试流式加载整个数据集并按扩展名过滤。
- **结果**: 效率极低，需要扫描海量元数据，速度仅为几十 Bytes/s。

### 2.2 文件夹结构与命名不一致
- **问题**: 直接加载文件夹时发现部分语言文件夹命名与预期不符（如 C# 实际为 `c-sharp` 而非 `csharp`）。
- **解决方案**: 编写调试脚本 `debug_files_v4.py` 遍历数据集根目录，确认了所有语言的正确文件夹名称，并建立了映射表。

### 2.3 下载速度优化
- **问题**: 默认的流式加载机制在处理大量文件时仍然较慢。
- **解决方案**:
  1. **直接 URL 加载**: 绕过 ModelScope SDK 的数据集加载逻辑，使用 `HubApi` 获取文件列表，构造直链（`https://modelscope.cn/.../resolve/master/...`）。
  2. **按需加载**: 发现每个 Parquet 文件约 300-500MB，对于 250MB 的目标，只需加载前 1-2 个文件即可，避免了全量解析 50+ 个文件的开销。
  3. **工具切换**: 使用 HuggingFace 的 `datasets` 库直接加载 Parquet URL，配合 `streaming=True` 实现高效流式读取。

## 7. 分层数据收集 (Stratified Data Collection)

### 7.1 需求变更
- **背景**: 原始数据中低质量（0-1 Star）代码占比过高 (>50%)，高质量代码稀缺。
- **新目标**: 按照 Star 数分层收集数据，确保高质量代码的比例。
- **分层标准**:
  - Tier 1 (>500 Stars): 20%
  - Tier 2 (100-500 Stars): 20%
  - Tier 3 (10-100 Stars): 30%
  - Tier 4 (<10 Stars): 30%

### 7.2 执行策略
- **脚本**: `src/collect_stratified_data.py`
- **逻辑**:
  - 针对每种语言设定目标大小。
  - 在每种语言内部，维护 4 个 Tier 的计数器。
  - 流式扫描数据集，根据 `max_stars_count` 将代码分配到对应的 Tier。
  - 当某个 Tier 满了之后，跳过该 Tier 的数据，继续寻找其他 Tier 的数据（主要是寻找稀缺的 Tier 1/2）。

### 7.3 结果
- **总耗时**: 约 43 分钟。
- **总数据量**: 1.10 GB。
- **分布验证**:
  - Tier 1 (>500): 220.19 MB (20.0%)
  - Tier 2 (100-500): 220.58 MB (20.0%)
  - Tier 3 (10-100): 330.03 MB (30.0%)
  - Tier 4 (<10): 330.77 MB (30.0%)
- **结论**: 完美达成预定分层目标。
